CSV结构
CSV文件的高级结构是：每一行表示一个实体（通常是一个国家或地区）和一个时间点（通常是一年）的观测值。

CSV文件的前两列是“Entity”和“Code”。“Entity”是实体名称（例如“United？States”）。“Code”是当实体是国家或地区时我们使用的OWID内部实体代码。对于大多数国家，这与实体的iso alpha-3代码相同（例如“USA”）——对于像历史国家这样的非标准实体，则使用自定义代码。

第三列是“Year”或“Day”。如果数据是年度数据，此列为“Year”，仅包含整数的年份。如果该列为“Day”，则包含格式为“YYYY-MM-DD”的日期字符串。

最后一列是数据列，即支撑图表的时间序列。如果CSV数据是使用“完整数据”选项下载的，则该列对应于下面的时间序列。如果CSV数据是使用“仅图表中可见的选定数据”选项下载的，则数据列会根据图表类型进行转换，因此与时间序列的关联可能不那么直接。

Metadata.json结构
.metadata.json文件包含有关数据包的元数据。“charts”键包含用于重新创建图表的信息，如标题、副标题等。“columns”键包含有关CSV中每一列的信息，如单位、覆盖的时间范围、数据的引用等信息。

关于这些数据您应该了解的信息
二氧化碳（CO₂）是导致气候变化的主要温室气体。

全球CO₂排放量十多年来一直保持在每人略低于五吨的水平。但各国之间的排放量差异很大，有些在上升，有些在下降。

化石燃料燃烧是CO₂排放的主要来源。该数据包括来自交通运输、发电和供暖等活动的化石CO₂排放量。

这些数字不包括土地利用变化（如森林砍伐或再造林）产生的CO₂排放量。

国际航空和航运的排放量不包含在任何单个国家或地区的数据中。它们仅计入全球总量中。

该数据基于领土排放量，即在一个国家边界内产生的排放量，但不包括来自进口商品的排放量。例如，进口钢材的排放量计入生产该钢材的国家。要了解更多并查看经贸易调整后的排放量，请阅读我们的文章：当我们根据贸易进行调整时，各国CO₂排放量如何比较？

数据来源于全球碳预算。化石CO₂排放量是使用关于能源使用（如煤炭、石油和天然气消费）以及工业生产（特别是水泥）的国家统计数据估算的。这些数据使用标准化排放因子被转换为CO₂排放量。有关更多详细信息，请阅读《全球碳预算》论文。

人均CO₂排放量是通过排放量除以人口计算得出的。它们代表一个国家或地区的人均平均排放量。要了解不同指标如何反映CO₂排放的分布情况，请阅读我们的文章：人均、国家、历史：各国在CO₂指标上如何比较？

关于此指标处理步骤的说明
使用因子3.664将全球排放量从吨碳转换为吨二氧化碳（CO₂）。这是全球碳项目建议的转换因子。它反映了一吨碳在完全氧化后，根据CO₂中碳和氧的相对分子量，会形成3.664吨CO₂。

1991年科威特油火的排放量包含在科威特当年的排放量中。

为了计算人均CO₂排放量，我们将原始数据除以一个国家/地区的估算人口。这些估算值来自我们基于多个来源的人口数据集。
